АБК можно рассматривать как способ введения интерфейса. АБК требует, чтобы его чистые виртуальные функции перегружались во всез конкретных производных классах - т.е. производных класс должен подчиняться правилам интерфейса.
**Чистая виртуальная функция** - способ представление нереализованной функции. В конце своего объявления содержит конструкцию `=0`:
```cpp
...
virtural double Area() const = 0;
...
```
Если объявление класса содержит чистую виртуальную функцию то объект такого класса создать невозможно.
Чтобы класс был АБК, он должен содержать, по крайней мере, одну виртуальную функцию.
Конструкция `=0` у виртуальной функции указывает на то, что класс является абстракнтым, и функцию в нём определять можно, но не обязательно.
